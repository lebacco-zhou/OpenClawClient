🐙 GitHub 推送指南

将 OpenClaw Client 推送到 GitHub 仓库

📌 快速导航：
  • 步骤 1: 创建 GitHub 仓库
  • 步骤 2: 推送代码
  • 步骤 3: 验证推送
  • 常见问题

📦 步骤 1: 在 GitHub 创建仓库

1 访问 GitHub 创建页面

打开浏览器访问：https://github.com/new

2 填写仓库信息

  • Repository name: OpenClawClient
  • Description: 安全加密的 Windows 桌面聊天客户端 - .NET 8 WPF
  • Public 或 Private（建议 Private，包含配置示例）
  • ❌ 不要勾选 "Add a README file"
  • ❌ 不要勾选 ".gitignore"
  • ❌ 不要勾选 "Choose a license"

3 点击 "Create repository"

🚀 步骤 2: 推送代码到 GitHub

💡 提示： 在服务器或本地计算机上执行以下命令

2.1 获取仓库地址

创建成功后，GitHub 会显示仓库地址，格式如下：

https://github.com/你的用户名/OpenClawClient.git

复制这个地址！

2.2 添加远程仓库并推送

# 进入项目目录
cd /root/.openclaw/workspace/OpenClawClient

# 添加远程仓库（替换为你的地址）
git remote add origin https://github.com/你的用户名/OpenClawClient.git

# 验证远程仓库
git remote -v

# 推送代码
git push -u origin master
✅ 推送完成后，访问你的 GitHub 仓库页面确认代码已上传：
https://github.com/你的用户名/OpenClawClient

✅ 步骤 3: 验证推送

访问 GitHub 仓库页面，确认以下内容：

  • ✅ 显示 7 次提交记录
  • ✅ 文件结构完整（src/、scripts/、docs/ 等）
  • ✅ README.md 内容正确显示

🔄 后续更新代码

首次推送后，后续更新更简单：

# 修改代码后
git add .
git commit -m "描述你的修改"
git push

📥 在其他设备克隆

在 Windows 上克隆项目：

# 创建项目目录
mkdir C:\Projects
cd C:\Projects

# 克隆项目
git clone https://github.com/你的用户名/OpenClawClient.git

# 进入项目目录
cd OpenClawClient

# 还原依赖
dotnet restore

# 运行
dotnet run --project src\OpenClawClient.Desktop

🔐 使用 SSH 方式（可选）

如果配置了 SSH 密钥，可以使用 SSH 地址：

# SSH 地址格式
git@github.com:你的用户名/OpenClawClient.git

# 添加远程仓库
git remote add origin git@github.com:你的用户名/OpenClawClient.git

# 推送
git push -u origin master

🐛 常见问题

Q: git remote add origin 报错 "remote origin already exists"

解决方法： 远程仓库已存在，先删除再添加： 
git remote remove origin
git remote add origin <新地址>

Q: git push 报错 "Permission denied"

解决方法：
  1. 检查 GitHub 用户名是否正确
  2. 确认有仓库写入权限
  3. 或使用 SSH 方式（需配置 SSH 密钥）

Q: 推送大文件失败

解决方法： GitHub 单文件限制 100MB，使用 Git LFS： 
git lfs install
git lfs track "*.zip"
git add .gitattributes
git commit -m "Configure LFS"
git push
